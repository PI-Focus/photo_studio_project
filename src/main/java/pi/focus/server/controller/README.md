## Пакет `controller/` — Web-слой

**Назначение:** Точка входа HTTP-запросов.

### Правила:
1.  **Тонкий контроллер:** Минимум логики. Только:
    *   Принять запрос (`@GetMapping`).
    *   Вызвать метод сервиса.
    *   Положить результат в `Model`.
    *   Вернуть имя view (шаблона).
2.  **Зависимость от абстракций:** Контроллер зависит от **Интерфейса Сервиса** (или Контекста), а не от конкретного класса мока.
3.  **Без знаний о Mocks:** Контроллер не должен импортировать классы из пакета `mocks`.
