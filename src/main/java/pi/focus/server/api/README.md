## Пакет `api/` — Контракты

**Назначение:** Строгое определение структуры данных, доступных в шаблонах.

### Правила:
*   Здесь находятся **ТОЛЬКО интерфейсы**.
*   Никакой бизнес-логики, никаких полей состояния, никаких реализаций.
*   Только геттеры (`getTitle()`, `getList()`).

### Подпакеты:
*   `api/context/`: Интерфейсы, описывающие полный набор данных для конкретной страницы (например, `IInfoContext` для главной страницы). Контроллер передает объект этого типа в модель Thymeleaf.
*   `api/models/`: Интерфейсы отдельных компонентов (карточек, табов, блоков), из которых состоят контексты.

> ⚠️ **Важно:** Любое изменение интерфейса в `api/` требует согласования с фронтендом.
